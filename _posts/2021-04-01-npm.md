# npm (Node Package Manager)

자바스크립트 패키지 매니저, nodeJS에서 사용할수있는 모듈들을 패키지화하여 모아둔 저장소 역할을 하며 설치/관리를 수행할수 있는 CLI를 제공한다.

### package.json

package.json은 프로젝트의 정보를 정의하고, 의존하는 패키지 버전 정보를 명시하는 파일.

Spring의 pom.xml과 같은 역할.

이 글에서는 기본적으로 알아야할 내용만 기술할 예정이고, 자세한 정보는 npm 문서에서 [package.json][https://docs.npmjs.com/cli/v7/configuring-npm/package-json] 확인가능.

```json
# Vue-cli로 생성한 프로젝트의 package.json 예시
{
  "name": "vue-practice", // 필수
  "version": "0.1.0",	// 필수
  "private": true, 
  "scripts": { // 해당 수명주기(key)에 실행될 명령어(value)
    "serve": "vue-cli-service serve",
    "build": "vue-cli-service build",
    "lint": "vue-cli-service lint"
  },
  "dependencies": { // 프로그램에 필요한 패키지
    "core-js": "^3.6.5",
    "vue": "^3.0.0"
  },
  "devDependencies": { // 로컬 개발 및 테스트에만 필요한 패키지
    "@vue/cli-plugin-babel": "~4.5.0",
    "@vue/cli-plugin-eslint": "~4.5.0",
    "@vue/cli-service": "~4.5.0",
    "@vue/compiler-sfc": "^3.0.0",
    "babel-eslint": "^10.1.0",
    "eslint": "^6.7.2",
    "eslint-plugin-vue": "^7.0.0"
  },
  "eslintConfig": { // eslint는 js 문법에서 에러를 찾아주는 도구.
    "root": true,
    "env": {
      "node": true
    },
    "extends": [
      "plugin:vue/vue3-essential",
      "eslint:recommended"
    ],
    "parserOptions": {
      "parser": "babel-eslint"
    },
    "rules": {}
  },
  "browserslist": [ // 지정된 브라우저와 호환되도록 js를 제어하는 도구
    "> 1%",
    "last 2 versions",
    "not dead"
  ]
}
```
